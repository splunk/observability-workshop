Database Monitoring lives alongside the rest of Splunk Observability Cloud's APM capabilities. We will open it now so the rest of the workshop has a familiar starting point.

{{% notice title="Exercise" style="green" icon="running" %}}

* From the left-hand navigation in Splunk Observability Cloud, click on **APM** **(1)**.
* In the APM landing page, find the **Database Monitoring** section and click **Overview** **(2)**.

<!-- TODO screenshot: Splunk Observability Cloud left nav with APM expanded and Database Monitoring → Overview highlighted with callouts (1) and (2) -->
![Navigate to Database Monitoring](../images/navigate-to-dbm.png)

* You should land on the **Database Monitoring Overview** page, which shows every database instance reporting telemetry into your org.

{{< tabs >}}
{{% tab title="Question" %}}
**Roughly how many database instances are visible on the overview page?**
{{% /tab %}}
{{% tab title="Answer" %}}
**It depends on the workshop environment** — the important thing is that instances from more than one database platform are listed side-by-side on the same page.
