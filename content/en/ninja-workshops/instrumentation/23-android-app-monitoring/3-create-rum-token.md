The Android agent needs a Splunk realm and a RUM access token. The RUM token authorizes
the mobile app to send user monitoring telemetry to Splunk Observability Cloud.

## Find Your Realm

Your realm is part of your Observability Cloud URL and ingest routing. Examples:

| UI URL | Realm |
| --- | --- |
| `https://app.us1.signalfx.com` | `us1` |
| `https://app.eu0.signalfx.com` | `eu0` |
| `https://app.au0.signalfx.com` | `au0` |

If your organization uses a custom or newer URL pattern, use the realm shown in your
Splunk Observability Cloud profile or onboarding page.

## Create or Identify a RUM Token

In Splunk Observability Cloud:

1. Open **Settings**.
2. Open **Access Tokens**.
3. Create or select a token that is allowed to send RUM data.
4. Copy the token value for the workshop.

Use a token dedicated to RUM and scoped for this app or workshop where possible. Do not
use a general API token in the Android app.

## Decide How the Token Enters the Build

For a workshop, a simple `gradle.properties` value is acceptable. For production,
inject the value through your build system and keep it out of source control.

Add these local-only values to your developer machine or CI secret store:

```properties
splunkRealm=us1
splunkRumAccessToken=replace-with-rum-token
splunkRumAppName=buttercup-android
splunkRumEnvironment=workshop
```

{{% notice title="Token Handling" style="warning" %}}
A RUM token is shipped to client devices, so it is not a secret in the same way a
server-side API token is. Still, keep it dedicated, rotate it when needed, and avoid
committing workshop or customer token values to the repository.
{{% /notice %}}

## Optional: Add BuildConfig Fields

If your app already uses `BuildConfig` for endpoint or environment values, add the RUM
values there. The exact syntax depends on your Gradle setup.

```kotlin
android {
    defaultConfig {
        buildConfigField("String", "SPLUNK_REALM", "\"${property("splunkRealm")}\"")
        buildConfigField("String", "SPLUNK_RUM_ACCESS_TOKEN", "\"${property("splunkRumAccessToken")}\"")
        buildConfigField("String", "SPLUNK_RUM_APP_NAME", "\"${property("splunkRumAppName")}\"")
        buildConfigField("String", "SPLUNK_RUM_ENVIRONMENT", "\"${property("splunkRumEnvironment")}\"")
    }
}
```

If your project does not expose `BuildConfig`, you can use Android resources, manifest
placeholders, or a small configuration wrapper that reads values generated by your
build.
