このワークショップでは、以下の内容を学びます

- Application Security のエントリーポイントへのナビゲーション方法。
- ランタイム脆弱性のインベントリ作成と、セキュリティ態勢をアプリケーションの健全性と相関させる方法。
- 脅威に基づくリスクスコアリングを使用して検出結果を優先順位付けする方法。
- 修復のガイド、脆弱性ステータスの管理、ライブラリの衛生状態のレビュー方法。
- ランタイム攻撃の調査と SecOps への通知統合の方法。

このワークショップでは、Application Security テレメトリ（ランタイム脆弱性、ライブラリインベントリ、攻撃イベント）を含む共有テナントが提供されます。

> *「テナントは APM でインストルメントされたマイクロサービスで事前構成されています — 既存の Observability インストルメンテーション以外に追加のエージェントをデプロイする必要はありません。」*

---

## 1.1 Splunk Observability Cloud へのアクセス

1. ブラウザを開き、**Splunk Workshop Realm** にサインインします（資格情報はワークショップのファシリテーターから提供されます）。
2. **Splunk Observability Cloud** のホーム画面が表示されることを確認します。
3. 左側のナビゲーションから **APM** を展開します - Application Security メニューオプションが表示されます。
4. ワークショップで使用する **environment** フィルターを確認します: `astronomy-shop-*` & 'ad'
