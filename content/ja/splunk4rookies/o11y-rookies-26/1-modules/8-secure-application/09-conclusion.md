## ワークショップの振り返り

チームが*本番環境で何が起きているのか、まずどこを確認すべきか？*と問いかけるとき、Observability は信頼性に関する問いに答えます。**Splunk Secure Application** は、その同じストーリーを**別のエージェントを追加したり、別の製品を使ったりすることなく**、アプリケーションセキュリティへと拡張します。

このワークショップでは、断片化されたアプリケーションセキュリティツールから、チームが本番環境を理解する方法にセキュリティが組み込まれた運用モデルへ移行する方法を学びました。以下の内容を扱いました

- **実行中の**アプリケーションにおける脆弱性と攻撃の検出
- 事後対応型の緊急対応から**プロアクティブな防御**への移行
- **既に運用している**計装を使用した、継続的な検出、ランタイム攻撃シグナル、およびサービス・環境・ライブラリに関連付けられた検出結果の活用

> *「その成果：セキュリティはアプリケーションを理解する方法の一部になります。実際にデプロイされているものを防御し、より安全に素早くリリースし、ノイズの追跡に費やす時間を削減できます。」*

## ワークショップモジュールのまとめ

| モジュール | 実践した機能 |
|--------|---------------------|
| **Runtime Vulnerability Inventory** | CVE、CVSS、ステータス、リスクスコアによる組織全体のエクスポージャービュー |
| **Maintaining Visibility** | Overview、Service Map、およびサービスごとの Application Security ワークスペース |
| **Prioritizing Known Threats** | 運用トリアージのための CVSS と Security Risk Score の活用 |
| **Updating and Upgrading** | 修復ガイダンス、影響範囲、サービスライブラリのコンテキスト |
| **Eliminating Technical Debt** | ステータスライフサイクルと組織全体のライブラリ衛生管理 |
| **Runtime Attack Investigation** | エクスプロイトフォレンジクスとコードレベルのスタックトレース |
| **Modernizing Defenses** | SIEM 通知と検出から適用へのロードマップ |
