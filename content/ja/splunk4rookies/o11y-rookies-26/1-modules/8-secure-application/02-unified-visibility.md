## 統合された可視性が重要な理由

信頼性とセキュリティが別々のツールに存在する場合、優先順位付けの議論が停滞します。SRE は*何が壊れたか？*と尋ね、AppSec は*何が悪用可能か？*と尋ねます。しかし、どちらのビューも、同時に不健全でリスクの高いサービスを表示しません。

Splunk Secure Application は、**APM Overview**、**Service Map**、およびサービスごとの **Application Security** ワークスペースにおいて、ゴールデンシグナルと並べて脆弱性と攻撃のサマリーを表示します。エンジニアリング、アプリケーションセキュリティ、および SecOps は、重複するエージェントやワークフローなしに、1 つのランタイムビューを共有できます。

---

## 2.1 APM Overview でのセキュリティポスチャー

1. **APM → Overview** に移動します。
2. **environment** フィルターを `astronomy-shop-*` に設定します。
3. **Services** タブまでスクロールします。

各サービスの行を確認します。標準的なヘルスメトリクスと並んで、インストルメント済みサービスのランタイム脆弱性と脅威プロファイルのサマリー（critical および high の CVE と攻撃の数）が表示されるはずです。

![apm](./images/02-overview.png)

> *「私たちはセキュリティと信頼性を統合しています。これにより、チームはアプリケーションのパフォーマンスと動作を理解するのと同じ場所で、Application Security のリスクを確認できます。」*

---

## 2.2 Service Map のランタイムセキュリティウィジェット

1. **APM → Service Map** に移動します。
2. **Services** フィルターを開き、**`ad`** を選択します。
3. Service Map の **`ad`** ノードをクリックします。
4. **Runtime Vulnerabilities** と **Attacks** ウィジェット（画面の右側）までスクロールします。

![apm](./images/02-servicemap.png)

ウィジェットには、上位の脆弱性（CVE タイトル、ID、スコア、ライブラリ）と攻撃アクティビティ（タイプと結果）がまとめて表示されます。

（オプション）- 脆弱性または攻撃の詳細（関連するウィジェットから）にドリルインして、ナビゲーションパスを確認します。

> *「このビューは爆発半径の考え方を強調しています。問題が依存関係やトラフィックの隣にフレーム化されています。」*
---

## 学んだこと

- APM Overview でサービスの健全性と脆弱性および脅威プロファイルを関連付ける方法。
- Service Map ウィジェットがトポロジーのコンテキストでセキュリティの問題をフレーム化する方法。
- サービスごとの Application Security が APM ワークスペース内でトリアージを維持する方法。
