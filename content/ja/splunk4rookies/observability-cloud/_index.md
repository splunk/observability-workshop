このワークショップでは、Splunk Observability Cloudがフロントエンドアプリケーションからバックエンドサービスまで、ユーザーエクスペリエンスへの即時的な可視性をいかに提供するかをデモンストレーションします。他のオブザーバビリティソリューションと一線を画す、プラットフォームの最も強力な機能のいくつかを探索する機会があります。

- **Infrastructure Monitoring**  
- **Full-fidelity Real User Monitoring (RUM)**  
- **NoSample Full-fidelity Application Performance Monitoring (APM)による完全なエンドツーエンドのトレース可視性**  
- **ノーコードのログクエリ**  
- **Synthetic Monitoring**  
- **タグ分析とエラースタックによる根本原因分析**  
- コンポーネント間をシームレスに移動できる **Related Content**

Splunk Observability Cloudの中核的な強みの1つは、テレメトリデータを統合し、エンドユーザーエクスペリエンスとアプリケーションスタック全体の包括的な全体像を作り上げる能力にあります。

ワークショップでは、AWS EC2インスタンス上にデプロイされたマイクロサービスベースのEコマースアプリケーションに焦点を当てます。ユーザーは商品を閲覧したり、カートに追加したり、購入を完了したりできます。このアプリケーションは、詳細なパフォーマンスデータを取得するためにOpenTelemetryで完全にインストルメントされています。

**OpenTelemetryとは?**  
OpenTelemetryは、メトリクス、トレース、ログなどのテレメトリデータを計装、生成、収集、エクスポートするためのツール、API、SDK (Software Development Kit) のオープンソースコレクションです。このデータにより、ソフトウェアのパフォーマンスと動作を詳細に分析できます。

OpenTelemetryコミュニティは、Splunk、Google、Microsoft、Amazonなどの主要企業に支えられ、急速に成長しています。現在、Cloud Native Computing Foundation内でKubernetesに次いで2番目に多くの貢献者を抱えています。
