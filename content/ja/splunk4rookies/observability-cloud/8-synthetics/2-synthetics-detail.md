現在、単一の Synthetic Browser Test の結果を見ています。このテストは **Business Transactions** に分割されており、これはビジネスクリティカルなユーザーフローを表す、論理的に関連する 1 つ以上のインタラクションのグループと考えてください。

{{% notice title="Info" style="info" %}}

下記のスクリーンショットにはエラーを示す赤いバナーが含まれていませんが、実行結果には表示される場合があります。これは、テスト実行が失敗するケースがあるためで想定された動作であり、ワークショップには影響しません。

{{% /notice %}}

![waterfall](../images/synth-waterfall.png)

1. **Filter:** テストの特定の部分にフォーカスします。filmstrip、video、および waterfall をフィルタリングします。
2. **Filmstrip:** サイトのパフォーマンスのスクリーンショット群を提供し、ページがリアルタイムでどのように応答するかを確認できます。
3. **Waterfall chart:** waterfall チャートは、テストランナーとテスト対象サイトのインタラクションを視覚的に表現したものです。
4. **Video:** 特定のテスト実行のロケーションとデバイスからサイトを読み込もうとするユーザーが、実際にどのような体験をするかを正確に確認できます。
5. **Browser test metrics:** ウェブサイトのパフォーマンスの全体像を提供するビューです。

デフォルトでは、Splunk Synthetics はテストのスクリーンショットとビデオキャプチャを提供します。これは問題のデバッグに役立ちます。たとえば、大きな画像の読み込みが遅い、ページのレンダリングが遅い、などを確認できます。

{{% exercise title="テスト再生を検査する" %}}

* マウスを使用して filmstrip を左右にスクロールし、テスト実行中にサイトがどのようにレンダリングされていたかを確認します。
* Video ペインで、再生ボタン **▶** を押してテスト再生を確認します。縦の三点リーダー **⋮** をクリックすると、*再生速度の変更*、*ピクチャーインピクチャー* での表示、ビデオの *ダウンロード* が可能です。
* filmstrip の上にあるフィルターで、*Synthetic Transactions* ヘッダーの下の **Home** をクリックします。
* 下の waterfall には、ページを構成するすべてのオブジェクトが表示されます。最初の行は HTML ページ自体です。次の行は、ページを構成するオブジェクト (HTML、CSS、JavaScript、画像、フォントなど) です。
* waterfall で **GET** *splunk-otel-web.js* の行を見つけます。
* **>** ボタンをクリックしてメタデータセクションを開き、Request/Response Header 情報を確認します。
* フィルターで、2 つ目の Synthetic Transaction である **Shop** に切り替えます。filmstrip と video が調整され、新しいトランザクションの先頭に移動することを確認します。
* 他のすべての Transaction についてもこれを繰り返し、最後に **PlaceOrder** トランザクションを選択します。

{{% /exercise %}}
