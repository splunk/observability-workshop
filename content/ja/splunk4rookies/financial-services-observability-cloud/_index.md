このワークショップでは、Splunk Observability Cloud が金融サービスのお客様にどのような価値を提供するかを実演します。インフラストラクチャからアプリケーション、ユーザーエクスペリエンスまで、デジタルエコシステム全体にわたってリアルタイムでフルフィデリティの AI を活用した監視を提供する能力が、その価値の源泉です。これは、最新のクラウドネイティブでマイクロサービスベースの環境向けに特別に構築されています。他のオブザーバビリティソリューションとは一線を画す、プラットフォームの最も強力な機能のいくつかを探求する機会があります。

- **Infrastructure Monitoring**
- **NoSample フルフィデリティ Application Performance Monitoring (APM) による完全なエンドツーエンドのトレース可視性**
- **ノーコードのログクエリ**
- **タグ分析とエラースタックによる根本原因分析**
- **コンポーネント間のシームレスなナビゲーションのための Related Content**

Splunk Observability Cloud の中核的な強みの1つは、テレメトリデータを統合し、エンドユーザーエクスペリエンスとアプリケーションスタック全体の包括的な全体像を作成する能力です。

このワークショップでは、Kubernetes 上にデプロイされたマイクロサービスベースの送金アプリケーションに焦点を当てます。ユーザーは送金を開始でき、ユーザー、残高、コンプライアンスに関するすべての適切なチェックが処理されます。このアプリケーションは、詳細なパフォーマンスデータをキャプチャするために OpenTelemetry で完全に計装されています。

**OpenTelemetry とは？**
OpenTelemetry は、メトリクス、トレース、ログなどのテレメトリデータの計装、生成、収集、エクスポートを支援するために設計された、オープンソースのツール、API、ソフトウェア開発キット（SDK）のコレクションです。このデータにより、ソフトウェアのパフォーマンスと動作を詳細に分析できます。

OpenTelemetry コミュニティは、Splunk、Google、Microsoft、Amazon などの大手企業に支えられ、急速に成長しています。現在、Cloud Native Computing Foundation 内で Kubernetes に次いで2番目に多くのコントリビューターを擁しています。
