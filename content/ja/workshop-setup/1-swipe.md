## **SWiPE を使用して組織を構成する**

> [!IMPORTANT] 重要
> **SWiPE** は EC2 インスタンスをプロビジョニング**しません**。これらのインスタンスは **Splunk Show** を使用して別途プロビジョニングされます。

**SWiPE** は、Splunk Observability Cloud でワークショップ環境を構成するためのオンラインツールです。**SWiPE** には[**こちら**](https://swipe.splunk.show)からアクセスできます。

![SWiPE](../images/swipe.png)

**SWiPE** はワークショップ環境に対して以下のタスクを実行します

{{< step "**ユーザーの作成と組織への招待**" "1" >}}

- メールアドレスを含む `.csv` ファイル（1行に1つ）をアップロードするか、メールアドレスを直接コピー＆ペースト（1行に1つ）します。
- **注意:** ユーザーには組織への参加招待メールが即座に送信されます。

{{< /step >}}

{{< step "**Ninja Workshop の管理者アクセス設定**" "2" >}}

- **Ninja Workshop** オプションが有効な場合、すべてのユーザーに **Admin** アクセス制御が付与されます。

{{< /step >}}

{{< step "**カスタム HEC URL とトークンの設定（オプション）**" "3" >}}

- **Override HEC URL and Token** が有効な場合、カスタムの HTTP Event Collector (HEC) URL とトークンを設定できます。これにより、Splunk Cloud へのログ送信に使用されるデフォルト値が上書きされます。

{{< /step >}}

{{< step "**チームの作成とユーザーの追加**" "4" >}}

- **SWiPE** がチームを作成し、すべてのユーザーを自動的に追加します。

{{< /step >}}

{{< step "**SWiPE ID の生成**" "5" >}}

- **SWiPE** がワークショップ用の一意の **SWiPE ID** を作成します。この ID をコピーして、[**Splunk Show**](https://show.splunk.com/home/) でワークショップインスタンスをプロビジョニングする際に使用する必要があります。

{{< /step >}}

> [!WARNING] 40名を超えるワークショップ
> ワークショップの参加者が **40名** を超える場合は、事前にサポートチームに連絡することをお勧めします。これにより、トライアルまたはワークショップ環境が負荷に対応できるよう適切にスケーリングされます。
