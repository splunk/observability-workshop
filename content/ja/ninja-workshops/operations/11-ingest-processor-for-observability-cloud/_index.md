インフラストラクチャやアプリケーション環境が極めて複雑になるにつれ、それらが生成するデータの量も大幅に増加し続けています。このデータ量と多様性の増加により、実用的なインサイトを得ることが難しくなり、問題の特定とトラブルシューティングの効率に影響を及ぼすことがあります。さらに、このデータを保存・アクセスするためのコストは急激に膨らむ可能性があります。多くのデータソース、特にログやイベントは、システム運用に対する重要な可視性を提供します。しかし、ほとんどの場合、効果的なモニタリングとアラートに実際に必要なのは、これらの膨大なログの中のごく一部の詳細だけです。

**よくある課題:**

* インフラストラクチャやアプリケーション環境の複雑化。
* これらの環境で生成されるデータ量の大幅な増加。
* 大量のデータから実用的なインサイトを得ることの難しさ。
* 膨大なデータの保存とアクセスに伴う高コスト。
* ログとイベントは重要な可視性を提供する一方で、本当に必要な情報はそのうちのごくわずかであることが多い。

これらの課題に対処するため、Splunk Ingest Processor は強力な新機能を提供します。それは、ログイベントをメトリクスに変換する機能です。メトリクスは保存と処理がより効率的で、問題のより迅速な特定を可能にし、その結果として Mean Time to Detection (MTTD) を短縮します。元のログやイベントを保持する必要がある場合は、S3 のような安価なストレージソリューションに保存することで、データ取り込みおよびそれを検索するために必要となる計算の全体的なコストを削減できます。

**ソリューション:**

* 可能な場合はログイベントをメトリクスに変換します。
* 必要に応じて、元のログやイベントを安価なストレージソリューションに保持します。
* 保持されたログへのアクセスと分析にフェデレーテッドサーチを活用します。

**アウトカム:**

* メトリクスは保存と処理がより効率的です。
* 問題のより迅速な特定により、Mean Time to Detection (MTTD) が短縮されます。
* データ取り込みと計算にかかる全体的なコストが低下します。
* モニタリングの効率とリソースの最適化が向上します。
* 運用コストを抑えながらシステム運用に対する高い可視性を維持できます。

このワークショップでは、Ingest Processor と Splunk Observability Cloud を実際に操作し、上記の課題に対処するためにどのように活用できるかを体験する機会があります。
