この _**テクニカル**_ Ingest Processor[^1] for Splunk Observability Cloud ワークショップでは、Splunk Enterprise Cloud の Ingest Processor をハンズオン形式で体験する機会を提供します。

ワークショップの各モジュールを簡素化するため、事前構成済みの Splunk Enterprise Cloud インスタンスが提供されます。

このインスタンスは、Ingest Processor パイプラインの作成に必要なすべての要件があらかじめ構成されています。

このワークショップでは、Ingest Processor を使用して詳細なログをメトリクスに変換し、それらのメトリクスを Splunk Observability Cloud に送信することの利点を紹介します。これらのテクニカルワークショップを終える頃には、Splunk Enterprise Cloud の Ingest Processor が持ついくつかの主要な機能や能力、そして Ingest Processor パイプラインの宛先として Splunk Observability Cloud を使用することの価値について十分な理解が得られます。

事前構成済みの [Splunk Enterprise Cloud](./1-access-cloud-instances/) インスタンスへのアクセス方法については、以下の手順をご覧ください。

![Splunk Ingest Processor Architecture](../images/IngestProcessor-architecture-diagram_release_updated2.png)

[^1]: [**Ingest Processor**](https://docs.splunk.com/Documentation/SplunkCloud/9.3.2408/IngestProcessor/AboutIngestProcessorSolution) は、Splunk Cloud Platform デプロイメント内で動作するデータ処理機能です。Ingest Processor を使用して、データフローを構成し、データ形式を制御し、インデックス作成前に変換ルールを適用し、宛先にルーティングできます。
