## コンテキストが重要です

前のセクションでは、生の Kubernetes 監査ログを確認し、それらをメトリクスに変換して Splunk Observability Cloud に送信する Ingest Processor Pipeline を作成しました。

このパイプラインが定義されたことで、Splunk Observability Cloud で新しいメトリクスを収集できるようになりました。これは素晴らしいスタートですが、現時点では特定の時間帯における Kubernetes 監査イベントの合計数を示す単一のメトリクスしか確認できません。ディメンションを追加することで、イベントタイプ、ユーザー、レスポンスステータスなどでメトリクスを分割できるようになり、はるかに価値のあるものになります。

このセクションでは、収集されているメトリクスに Kubernetes 監査ログから追加のディメンションを含めるよう、Ingest Processor Pipeline を更新します。これにより、監査ログの特定の側面に対してフィルタリング、グループ化、可視化、アラートをさらに行えるようになります。メトリクスを更新した後、ログに関連するさまざまなタイプのアクションのステータスを示す新しいダッシュボードを作成します。
