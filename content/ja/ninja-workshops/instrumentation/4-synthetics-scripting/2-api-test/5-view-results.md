## 結果の確認

テストがプロビジョニングされ、最初の実行がディスパッチされるまで数分待ちます。テストダッシュボードに少なくとも1回の成功実行が表示されたら、それをクリックして実行結果を詳しく確認します。

![API test result page showing the two-step Spotify transaction with response headers and request timing breakdown](../../img/api-test-result.png)

API テストの結果ページは、Real Browser Test の結果ページを簡素化したようなレイアウトになっています。基本的な構成は同じですが、スクリーンショット、動画、Web Vitals はありません（ブラウザを起動していないため、いずれも該当しないからです）。

## 結果ヘッダー

ページの上部には、**Status**、**Location**、**Timestamp**、**Total duration** が表示されています。RBT と比較して合計実行時間が非常に短いことに注目してください。複数ステップのトランザクション全体で通常は数十〜数百ミリ秒程度です。これが API テストのコスト・頻度面における優位性を示しています。

## リクエストタイムライン

トランザクション内の各リクエストが、実行順に水平バーで表示されます。緑色は成功したステップ（検証ルールも満たした 2xx レスポンス）、赤色は失敗を示します。バーの長さは各リクエストの合計実行時間に比例するため、チェーンの中でどのステップが遅いのかを一目で把握できます。

## リクエスト詳細ペイン

タイムライン内（または左側のリスト）で任意のリクエストをクリックすると、3つのタブが開きます。

- **Response** — レスポンスボディ、ヘッダー、ステータスコードの全体です。スポット的なデバッグ（「Spotify は実際に何を返しているのか？」）や、抽出ルールが正しいフィールドを指しているかの確認に役立ちます。レスポンスボディは毎回キャプチャされるため、挙動が変化した場合は今日のレスポンスと先週のレスポンスを比較することもできます。
- **Request** — 実際にネットワーク上に送信された内容そのものです。メソッド、URL（グローバル変数はすでに展開された状態で表示されるので置換結果を確認できます）、ヘッダー、ボディが含まれます。Concealed 変数による機密値は、ここでも引き続き伏せられた状態のままです。
- **Connection info** — TLS バージョン、暗号スイート、証明書チェーンが表示されます。証明書やプロトコルハンドシェイクの問題をデバッグするときに確認する価値があります。

## Request info — タイミングの内訳

右側のサイドバーは、このページで最も有用な診断ビューです。選択したリクエストについて、合計実行時間を構成フェーズごとに分解して表示します。

- **DNS time** — ホスト名の名前解決にかかった時間です。ここで急激なスパイクが出る場合、通常は地域 DNS プロバイダの問題を示しています。
- **TCP Connect time** — サーバーへの TCP ソケットを開くのにかかった時間です。急激なスパイクは通常、ネットワークの輻輳や対象サーバーの負荷を示しています。
- **TLS time** — TLS ハンドシェイクの完了にかかった時間です。ここでのスパイクは、証明書チェーンの問題や TLS 再ネゴシエーションの繰り返しを示している可能性があります。
- **Wait time** — リクエストが完全に送信されてからレスポンスの最初のバイトが返ってくるまでのサーバー処理時間です。これは**サーバー側のレイテンシ**であり、通常はページ上で最もアクションに結びつけやすい数値です。ここでリグレッションが発生した場合、API チームに直接問題があることを示します。
- **Receive time** — ネットワークからレスポンスボディを読み取るのにかかった時間です。レスポンスペイロードが大きい場合に大きくなります。
- **Duration** — この単一リクエストの合計ウォールクロック時間です。
- **Response Size** — 受信したバイト数です。予期しない増加に注意してください。レスポンスを肥大化させるスキーマ変更は、わかりにくいレイテンシリグレッションの一般的な原因です。

これらはいずれも、チャート化、ダッシュボード化、アラート化が可能な時系列データでもあります。一般的な本番環境のセットアップでは、(a) 実行全体の失敗、(b) 実行全体の所要時間 P95、(c) バックエンド SLO が重要なリクエストの Wait time に対してディテクタを作成します。
