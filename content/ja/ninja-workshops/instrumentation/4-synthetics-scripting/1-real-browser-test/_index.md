## はじめに

このワークショップでは、[**Chrome DevTools Recorder**](https://developer.chrome.com/docs/devtools/recorder/) を使用して、Splunk のデモインスタンスに対する Synthetic トランザクションを作成する方法を説明します。記録した内容を JSON としてエクスポートし、それをもとに **Splunk Synthetic Monitoring Real Browser Test** を作成します。このテストはクラウド上で、スケジュールに従って、世界中の複数のロケーションから実行されます。

### Real Browser Test とは

Real Browser Test (RBT) は、Splunk Synthetic Monitoring がグローバルクラウドロケーションのいずれかで **完全な Chromium ブラウザ** を起動し、サイトをロードして、クリック・フォーム入力・ナビゲーションといった一連のユーザー操作を、実際の人間と同じように実行する Synthetic チェックです。本物のブラウザを使うため、JavaScript を実行し、サードパーティの依存関係グラフ全体を読み込み、CSS を適用し、通常のユーザーのブラウザが行うあらゆる動作を捕捉します。これにより、ログイン、チェックアウト、検索、ダッシュボードのロードといったユーザージャーニーをエンドツーエンドで監視するのに最適です。

シンプルな HTTP アップタイムチェックとのトレードオフは、コストと実行頻度です。Real Browser での実行には 10〜30 秒かかり、より多くのキャパシティを消費するため、RBT は通常、ロケーションごとに 1〜5 分ごとにスケジュールされます。一方、アップタイムチェックは多数のエンドポイントに対して低コストで毎分実行できます。

### なぜ Chrome DevTools Recorder から始めるのか

Splunk の UI 上で Synthetic テストをステップごとに手動で作成することもできますが、まずジャーニーを記録するほうが速く、ミスも少なくなります。Recorder はクリックする各要素について、複数の堅牢なセレクター (CSS、XPath など) を捕捉するため、開発者がクラス名を変更したときにテストが最初の機会で壊れる可能性を低減できます。エクスポートされた JSON はバージョン管理にチェックインしたり、チームメンバーと共有したり、同じ記録フォーマットを扱う他のツールに取り込んだりできるポータブルな成果物でもあります。
