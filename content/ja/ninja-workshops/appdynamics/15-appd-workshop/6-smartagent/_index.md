## はじめに

AppDynamics **Smart Agent** は、インフラストラクチャに対して包括的な監視機能を提供する軽量でインテリジェントなエージェントです。このセクションでは、4つの異なるデプロイメントアプローチについて説明します。組織のニーズや既存のツールに最適な方法を選択できます。

![AppDynamics](https://img.shields.io/badge/AppDynamics-0078D4?style=flat)

## Smart Agent とは

Smart Agent は AppDynamics の次世代監視エージェントであり、以下の機能を提供します

- **統合監視**: インフラストラクチャ、アプリケーション、サービスを単一エージェントで監視
- **軽量設計**: リソースフットプリントを最小限に抑制
- **自動検出**: アプリケーションを自動的に検出して監視
- **ネイティブインストルメンテーション**: アプリケーションパフォーマンスの詳細な可視化
- **柔軟なデプロイメント**: 複数のインストールおよび管理オプション

## デプロイメントアプローチ

このセクションでは、Smart Agent を大規模にデプロイするための4つの異なるアプローチについて説明します

### 1. リモートインストール (smartagentctl)

SSH 経由でデプロイするために `smartagentctl` CLI ツールを使用する最も直接的なアプローチです。

**最適な用途**

- 中程度の数のホストへの迅速なデプロイメント
- 既存の CI/CD インフラストラクチャがない環境
- テストおよび概念実証シナリオ
- デプロイメントプロセスの直接制御

**主な機能**

- SSH ベースの直接デプロイメント
- シンプルな YAML 設定
- 追加ツール不要
- 並行実行サポート

### 2. Jenkins 自動化

Jenkins パイプラインを使用した完全なライフサイクル管理のためのエンタープライズグレードのデプロイメントです。

**最適な用途**

- すでに Jenkins を使用している組織
- 複雑なデプロイメントワークフロー
- 承認ゲートが必要な環境
- 既存の CI/CD パイプラインとの統合

**主な機能**

- パラメータ化されたパイプライン
- 数千台のホストのバッチ処理
- 完全なライフサイクル管理
- 一元化されたログとレポート

### 3. GitHub Actions 自動化

セルフホストランナーを使用した GitHub Actions ワークフローによる最新の CI/CD アプローチです。

**最適な用途**

- バージョン管理に GitHub を使用しているチーム
- クラウドネイティブ環境
- GitOps ワークフロー
- Web ベースの管理を好む分散チーム

**主な機能**

- 11 の専用ワークフロー
- VPC 内のセルフホストランナー
- GitHub secrets との統合
- スケーラビリティのための自動バッチ処理

### 4. Ansible 自動化

Infrastructure as Code のための Ansible Playbook を使用した構成管理アプローチです。

**最適な用途**

- 構成管理に Ansible を使用しているチーム
- 宣言的なインフラストラクチャ定義
- フリート全体の一貫した状態管理

**主な機能**

- Infrastructure as Code (IaC)
- 冪等な Playbook
- インベントリ管理
- ロールベースの構成

## 適切なアプローチの選択

| 要素 | リモートインストール | Jenkins | GitHub Actions | Ansible |
|--------|-------------------|---------|----------------|---------|
| **セットアップの複雑さ** | 低 | 中 | 中 | 低 |
| **スケーラビリティ** | 良好 (数百台) | 優秀 (数千台) | 優秀 (数千台) | 優秀 (数千台) |
| **前提条件** | SSH アクセスのみ | Jenkins サーバー | GitHub アカウント | Ansible Control Node |
| **学習コスト** | 最小限 | 中程度 | 中程度 | 低〜中程度 |
| **自動化レベル** | 手動実行 | 完全自動化 | 完全自動化 | 完全自動化 |
| **最適なユースケース** | 迅速なデプロイメント | エンタープライズ CI/CD | 最新の DevOps | Infrastructure as Code |

## すべてのアプローチに共通する機能

どのデプロイメント方法を選択しても、すべてのアプローチで以下が提供されます

- ✅ **SSH ベースのデプロイメント** - リモートホストへの展開
- ✅ **並行実行** - より高速なデプロイメント
- ✅ **完全なライフサイクル管理** (インストール、開始、停止、アンインストール)
- ✅ **構成管理** - コントローラー設定の管理
- ✅ **エラーハンドリング** とログ記録
- ✅ **スケーラビリティ** - 数百〜数千台のホストに対応

## ワークショップの構成

各デプロイメントアプローチには専用のセクションがあります

1. **リモートインストール** - CLI ベースの直接デプロイメント
2. **Jenkins 自動化** - パイプラインベースのエンタープライズデプロイメント
3. **GitHub Actions** - 最新のワークフローベースのデプロイメント
4. **Ansible 自動化** - Infrastructure as Code デプロイメント

ニーズに応じて、1つまたはすべてのアプローチに従うことができます。

{{% notice title="ヒント" style="primary" icon="lightbulb" %}}
Smart Agent のデプロイメントが初めての場合は、より自動化されたソリューションに移行する前に、基本を理解するために **リモートインストール** アプローチから始めることをお勧めします。
{{% /notice %}}

## 前提条件

いずれかのデプロイメントアプローチに進む前に、以下を確認してください

- コントローラーアクセス権を持つ AppDynamics アカウント
- アカウント名とアクセスキー
- SSH アクセスが可能なターゲットホスト
- ホストから AppDynamics Controller へのネットワーク接続
- ターゲットホストでの適切な権限

## 次のステップ

お好みのデプロイメントアプローチを選択して、そのセクションに進んでください

- **シンプルに始める**: リモートインストールから基本を学びます
- **Jenkins でスケール**: エンタープライズグレードの自動化には Jenkins に進みます
- **GitHub で最新化**: クラウドネイティブワークフローには GitHub Actions を採用します
- **Ansible で自動化**: 宣言的な構成管理には Ansible を使用します

各セクションでは、Smart Agent を大規模にデプロイするための完全なハンズオンガイダンスを提供します！
