一部のリクエストでエラーやパフォーマンス低下が発生する理由を理解するには、トレースにコンテキストを追加する必要があります。これはタグを追加することで実現します。しかしまず、タグとは何か、そしてなぜオブザーバビリティにとってタグが重要なのかについて説明します。

## タグとは？

タグは、トレース内のスパンに関する追加のメタデータを提供するキーと値のペアであり、Splunk APM に送信するスパンのコンテキストを充実させることができます。

たとえば、決済処理アプリケーションでは、以下を追跡できると便利です

* 使用された支払い方法（クレジットカード、ギフトカードなど）
* 支払いをリクエストした顧客の ID

このようにしておくことで、支払い処理中にエラーやパフォーマンスの問題が発生した場合に、トラブルシューティングに必要なコンテキストを得ることができます。

一部のタグは OpenTelemetry collector で追加できますが、このワークショップで扱うタグはより細かい粒度のもので、アプリケーション開発者が OpenTelemetry API を使用して追加します。

## Attributes と Tags

先に進む前に、用語について補足します。このワークショップでは**タグ**について扱っており、これは **Splunk Observability Cloud** で使用される用語ですが、OpenTelemetry では代わりに **attributes** という用語を使用します。したがって、このワークショップでタグという言葉が出てきた場合は、attributes と同義として扱ってください。

## なぜタグがそれほど重要なのか？

タグは、アプリケーションが真にオブザーバブルであるために不可欠です。クレジットチェックサービスで見たように、一部のユーザーは高速でエラーのない優れた体験をしています。しかし、他のユーザーは遅い体験をしたり、エラーに遭遇したりします。

タグはトレースにコンテキストを追加し、なぜ一部のユーザーが優れた体験を得て、他のユーザーはそうでないのかを理解するのに役立ちます。そして、**Splunk Observability Cloud** の強力な機能はタグを活用して、根本原因に素早くたどり着くことを支援します。

## プレビュー：Tag Spotlight

**Tag Spotlight** はタグを使用して、高レイテンシーやエラー率の原因となるトレンドを発見します

![Tag Spotlight Preview](../images/tag_spotlight_preview.png)

上のスクリーンショットは、別のアプリケーションにおける Tag Spotlight の例を示しています。

Splunk は、決済サービスに関連するトレースの一部として含まれるすべてのタグを分析しています。

これにより、一部のタグ値が他のものよりもエラーが多いかどうかを素早く把握できます。

version タグを見ると、サービスのバージョン `350.10` は 100% のエラー率であるのに対し、バージョン `350.9` にはエラーがまったくないことがわかります
