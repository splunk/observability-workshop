一部のリクエストでエラーやパフォーマンス低下が発生する理由を理解するには、トレースにコンテキストを追加する必要があります。これはタグを追加することで実現します。まず、タグとは何か、そしてなぜオブザーバビリティにとってタグが重要なのかについて説明します。

## タグとは？

タグは、トレース内のスパンに追加のメタデータを提供するキーと値のペアであり、Splunk APM に送信するスパンのコンテキストを充実させることができます。

例えば、決済処理アプリケーションでは、以下の情報を追跡すると便利です

* 使用された支払い方法（クレジットカード、ギフトカードなど）
* 支払いをリクエストした顧客の ID

これにより、決済処理中にエラーやパフォーマンスの問題が発生した場合に、トラブルシューティングに必要なコンテキストを得ることができます。

一部のタグは OpenTelemetry collector で追加できますが、このワークショップで扱うタグはより粒度が細かく、アプリケーション開発者が OpenTelemetry API を使用して追加するものです。

## Attributes と Tags

先に進む前に用語について補足します。このワークショップでは **tags** について説明しており、これは **Splunk Observability Cloud** で使用される用語ですが、OpenTelemetry では代わりに **attributes** という用語を使用します。そのため、このワークショップで tags という言葉が出てきた場合、attributes と同義として扱ってください。

## なぜタグはそれほど重要なのか？

タグは、アプリケーションを真にオブザーバブルにするために不可欠です。クレジットチェックサービスで確認したように、一部のユーザーはエラーのない高速な優れた体験をしています。しかし、他のユーザーは遅い体験をしたり、エラーに遭遇したりします。

タグはトレースにコンテキストを追加し、なぜ一部のユーザーは優れた体験を得られ、他のユーザーはそうでないのかを理解するのに役立ちます。また、**Splunk Observability Cloud** の強力な機能はタグを活用して、根本原因に素早くたどり着けるようにします。

## プレビュー: Tag Spotlight

**Tag Spotlight** はタグを使用して、高レイテンシーやエラー率に寄与するトレンドを発見します

![Tag Spotlight Preview](../images/tag_spotlight_preview.png)

上のスクリーンショットは、別のアプリケーションでの Tag Spotlight の例を示しています。

Splunk は、決済サービスに関連するトレースの一部として含まれるすべてのタグを分析しています。

これにより、一部のタグ値が他よりも多くのエラーを持っているかどうかを非常に素早く確認できます。

version タグを見ると、サービスのバージョン 350.10 は 100% のエラー率であるのに対し、バージョン 350.9 にはまったくエラーがないことがわかります

![Tag Spotlight Preview](../images/tag_spotlight_preview_details.png)

独自のタグをキャプチャした後、ワークショップの後半でクレジットチェックサービスに対して Tag Spotlight を使用します。
