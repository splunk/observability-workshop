このワークショップでは、以下のことを達成しました

* アプリケーションをデプロイし、**Splunk Observability Cloud** でトレースをキャプチャしました。
* **Database Query Performance** を使用して、ゲームの起動時間に影響を与えている低速クエリを特定しました。
* **AlwaysOn Profiling** を有効にし、起動時間の増加とメモリ使用量の原因となっているコード行を確認しました。
* 別のアプリケーションパフォーマンスの問題を発見し、再び **AlwaysOn Profiling** を使用して問題のあるコード行を特定しました。
* これらの問題の両方に修正を適用し、**Splunk Observability Cloud** を使用して結果を検証しました。

アプリケーションで **AlwaysOn Profiling** を有効にし、**Database Query Performance** を活用することで、**Splunk Observability Cloud** に送信しているデータからさらに多くの価値を得ることができます。  

このワークショップを完了した今、あなた自身のアプリケーションからより深い診断データの収集を開始するための知識を身につけました！

**Database Query Performance** を今すぐ始めるには、[Monitor Database Query Performance](https://docs.splunk.com/observability/en/apm/db-query-perf/db-query-performance.html) をご覧ください。

**AlwaysOn Profiling** を始めるには、[Introduction to AlwaysOn Profiling for Splunk APM](https://docs.splunk.com/observability/en/apm/profiling/intro-profiling.html) をご覧ください。
