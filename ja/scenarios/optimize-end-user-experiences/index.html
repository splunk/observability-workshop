エンドユーザーエクスペリエンスの最適化

Splunk RUM と Synthetics を組み合わせて、実際のユーザーパフォーマンスを測定し、プロアクティブなテストを実行し、重要なフロントエンド KPI にアラートを設定します。

90 minutes Author Sarah Ware

Splunk Observability を使用して、エンドユーザーエクスペリエンスに関するインサイトを取得し、エクスペリエンスを改善するためのシナリオをプロアクティブにテストするにはどうすればよいでしょうか？

セクション:

  • 可用性とパフォーマンスをすぐに把握するための基本的な Synthetic テスト を設定する
    • Uptime テスト
    • API テスト
    • シングルページ Browser テスト
  • RUM を探索して実際のユーザーを理解する
  • ユーザーについて学んだことと、ユーザーに必要な操作に基づいて高度な Synthetics テスト を作成する
  • KPI をキャプチャし、トレンドを表示し、イベントのコンテキストでデータを表示するためのダッシュボードチャート をカスタマイズする
  • KPI にアラートを設定するための Detector を作成する

Tip

ワークショップ全体を通じて次のことを念頭に置いてください：エンドユーザーと自分自身／開発者にとって、最も早く価値を得るために、どのように戦略的にアクティビティの優先順位を付けることができるでしょうか？

コンテキスト #

フロントエンドパフォーマンスモニタリングは、エンドユーザーエクスペリエンスに関わるすべてをキャプチャするために必要です。バックエンドだけを監視している場合、ユーザーの成功に不可欠な他のすべてのリソースを見逃していることになります。実際の事例については What the Fastly Outage Can Teach Us About Observability をお読みください。下の画像をクリックすると拡大できます。

参考資料 #

このワークショップでは、エンドユーザーエクスペリエンスをさらに理解し、最適化する方法に関するリソースへの参照が随所に登場します。サポートされている機能については Splunk Docs 、ヒントやコツについては Lantern に加えて、Google’s web.dev や Mozilla も優れたリソースです。

使用している特定のライブラリ、プラットフォーム、CDN には、それぞれ独自のリソースがあることも覚えておいてください。例えば、React 、Wordpress 、Cloudflare にはそれぞれパフォーマンスを改善するための独自のヒントがあります。
