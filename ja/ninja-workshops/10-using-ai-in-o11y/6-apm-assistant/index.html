APM AI Assistant とインテリジェントトラブルシューティング

7 min

APM AI Assistant とは？ #

APM AI Assistantは、コンテキストに応じたガイダンスの提供、トレースの分析、調査中の次のステップの提案により、アプリケーションパフォーマンスの問題のトラブルシューティングを支援するインテリジェントな機能です。APMデータを理解し、解決策に導く仮想エキスパートとして機能します。

Note

AI Assistantの機能は、Splunk Observability Cloudのバージョンとライセンスによって異なる場合があります。ここで説明する機能の一部は、プレビュー段階であるか、特定のライセンスが必要な場合があります。

主な機能 #

1. トレース分析 #

  • 自動スパン分析: 遅いスパンや問題のあるスパンを特定します
  • ボトルネック検出: 分散トレースにおけるパフォーマンスのボトルネックをハイライトします
  • エラーパターン認識: エラートレースをグループ化して分析します
  • 依存関係のインサイト: サービスの依存関係と呼び出しパターンを理解します

2. ガイド付きトラブルシューティング #

  • 根本原因の提案: トレースデータに基づいて可能性の高い根本原因を提案します
  • 調査の経路: 次に調べるべき項目を提案します
  • 過去との比較: 現在の問題を過去のパターンと比較します
  • 解決策の推奨: 類似の問題に基づいて潜在的な修正を提案します

3. コンテキストに応じたインサイト #

  • 自然言語による要約: 複雑なトレースを平易な言葉で説明します
  • 影響評価: 問題の範囲と深刻度を推定します
  • サービスヘルスのインサイト: サービスパフォーマンスの傾向を要約します
  • 異常の説明: なぜそれが異常と見なされるかを説明します

AI Assistant がどのように役立つか #

シナリオ 1: 遅いトレースの調査 #

従来のアプローチ:

  1. トレースウォーターフォールを開く
  2. すべてのスパンを手動でスキャン
  3. 所要時間を計算
  4. 最も遅い操作を特定
  5. 他のトレースと相互参照
  6. 根本原因についての仮説を立てる

AI Assistant を使用した場合:

  1. トレースを開く
  2. AIがハイライト: “Database query in checkout-service took 2.3s (95th percentile: 45ms)”
  3. 提案: “Check database index on orders table”
  4. 同じパターンの類似トレースにリンク
  5. パターンがいつ始まったかを表示

シナリオ 2: エラーパターンの理解 #

AI Assistant が提供するもの:

  • 類似エラーのグループ化
  • 発生頻度の分析
  • 最初の発生タイムスタンプ
  • 影響を受けたエンドポイントとサービス
  • エラートレース全体に共通する属性
  • 推奨される調査手順

ハンズオン演習: AI を活用した APM 機能の使用 #

Exercise

ステップ 1: サービスインサイトの探索 #

  1. APM → Services に移動します
  2. パフォーマンスデータのあるサービスを選択します
  3. AIが生成したインサイトや要約を探します:
    • サービスヘルススコア
    • パフォーマンスの傾向
    • 異常インジケーター
    • 主な問題やボトルネック

ステップ 2: AI アシスタンスを使用したトレースの分析 #

  1. APM → Traces に移動します
  2. 遅いトレースまたはエラートレースでフィルタリングします
  3. トレースウォーターフォールビューを開きます
  4. AIを活用した機能を探します:
    • ハイライトされた問題のあるスパン
    • 自動クリティカルパス特定
    • ベースライントレースとの比較
    • 提案された根本原因

ステップ 3: 自動根本原因検出の活用 #

  1. トレースビューで Root Cause または Insights パネルを見つけます
  2. AIの提案を確認します:
    • どのスパンがボトルネックか？
    • 通常の動作と比較して何が変わったか？
    • どのタグや属性が問題と相関しているか？
  3. 提案された調査パスに従います
  4. 特定されたコンポーネントをドリルダウンします

ステップ 4: トレース比較の使用 #

  1. 問題のあるトレースを選択します
  2. Compare または Similar Traces 機能を探します
  3. AIが表示するもの:
    • 類似の正常なトレース（ベースライン）
    • 遅いトレースで何が異なるか
    • 統計的な比較
  4. 異常なコンポーネントを特定します

インテリジェントトレース機能 #

クリティカルパスのハイライト #

AIは分散トレースを通じてクリティカルパスを自動的に特定します:

  • クリティカルスパン: 合計レイテンシに直接寄与したスパン
  • 並列化可能なスパン: 非同期処理で最適化できるスパン
  • 待機時間: ダウンストリームサービスを待機している時間

スパン異常検出 #

AIは以下を考慮して異常なスパンを検出します:

  • 所要時間: 過去のベースラインとの比較
  • 頻度: このスパンがどのくらい頻繁に出現するか
  • エラー率: このスパンのエラー vs通常時
  • コンテキスト: 通常とは異なるタグや属性

サービス依存関係のインテリジェンス #

AIはサービスアーキテクチャを理解します:

  • 依存関係マッピング: サービス間の関係を自動的にマッピング
  • 影響分析: サービスの問題が依存先にどのように影響するかを予測
  • 循環依存の検出: 問題のある呼び出しパターンを特定
  • 最適化の提案: アーキテクチャの改善を推奨

AI を活用した APM アラート #

スマートアラート優先順位付け #

AIは以下によりアラートの優先順位付けを支援します:

  • ビジネス影響スコアリング: ユーザー/収益への影響を推定
  • 過去のコンテキスト: 過去の類似アラートと比較
  • 相関分析: 関連するアラートをグループ化
  • ノイズ削減: 誤検知の可能性が高いものを抑制

適応型しきい値 #

APMベースのディテクター向け:

  • 動的ベースライン: トラフィックパターンに基づいてしきい値を調整
  • 季節性の認識: 時間帯/曜日のパターンを考慮
  • デプロイメントの認識: デプロイメントイベントを認識
  • トラフィック比例アラート: トラフィック量の変化に応じて調整

自然言語機能 #

質問する（利用可能な場合） #

一部のAI Assistant実装では、自然言語クエリが可能です:

質問の例:

  • “Why is checkout-service slow?”
  • “What changed in the last hour?”
  • “Which endpoints are experiencing errors?”
  • “Show me traces from customer tier enterprise”
  • “Compare current performance with last week”

AI が提供するもの:

  • 自然言語での回答
  • 関連するトレースとメトリクス
  • データの可視化
  • 推奨される次のステップ

AI Assistant のベストプラクティス #

1. 豊富なコンテキストを提供する #

AIがあなたを助けられるようにしましょう:

  • 説明的なスパン名を使用する
  • 関連するタグと属性を追加する
  • スパンにエラーの詳細を含める
  • デプロイメントイベントにタグを付ける

2. 信頼しつつ検証する #

  • AIの提案を出発点として使用する
  • 実際のデータで発見を検証する
  • メトリクスとログと相互参照する
  • ドメイン知識を適用する

3. AI パターンから学ぶ #

  • AIが特定する一般的な根本原因に注目する
  • どのタグが最も有用かを観察する
  • AIが提案する調査パスを研究する
  • 繰り返しパターンに基づいて自動化を構築する

4. フィードバックを提供する #

AI Assistantがフィードバックをサポートしている場合:

  • 役立った提案をマークする
  • 不正確な分析を報告する
  • システムはあなたのフィードバックから学習します

AI Assistant と他の AI 機能の組み合わせ #

統合ワークフロー #

  1. アラート発火 (AutoDetect MLディテクター)
  2. Tag Spotlight で問題を絞り込む
  3. APM AI Assistant が影響を受けたトレースを分析
  4. Related Content が関連ダッシュボードを表示
  5. Log Observer AI が相関するログパターンを表示
  6. 完全なコンテキストで解決

調査フローの例 #

text 
Alert: "Latency increased on payment-service"
   ↓
Tag Spotlight: "Region: us-west-1 (87% contribution)"
   ↓
APM AI: "Database span duration increased 450%"
   ↓
Trace Analysis: "Connection pool exhausted"
   ↓
Log Observer AI: Pattern "Connection pool timeout" increased
   ↓
Related Content: Dashboard "Database Connection Health"
   ↓
Root Cause: Recent traffic spike exceeded DB connection limits

制限事項と考慮事項 #

  • 学習期間: AIは比較のために過去のデータが必要です
  • データ品質: 精度はトレースの完全性とタグ付けに依存します
  • コンテキストの境界: AIはあなたのビジネスロジックを知りません
  • プレビュー機能: 一部の機能は進化中の可能性があります
  • プライバシー: 機密データがトレース属性に含まれないようにしてください

Tip

APM AI Assistantは、アプリケーションが包括的なタグと属性で適切に計装されている場合に最も効果的です。トレースデータが豊富であればあるほど、AIのインサイトも優れたものになります。
