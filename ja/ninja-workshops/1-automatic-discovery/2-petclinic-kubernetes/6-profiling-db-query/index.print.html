Trace WaterfallеҶ…гҒ®Always-On Profiling

APM Waterfallгғ“гғҘгғјгҒ§гӮӘгғӘгӮёгғҠгғ«гҒ®Trace & Span (1)пјҲгҒҫгҒҹгҒҜйЎһдјјгҒ®гӮӮгҒ®пјүгӮ’йҒёжҠһгҒ—гҖҒеҸіеҒҙгҒ®гғҡгӮӨгғігҒӢгӮү**Memory Stack Traces (2)**гӮ’йҒёжҠһгҒ—гҒҰгҒҸгҒ гҒ•гҒ„

гғҡгӮӨгғігҒ«Memory Stack Trace Flame Graph **(3)**гҒҢиЎЁзӨәгҒ•гӮҢгҒҫгҒҷгҖӮгӮ№гӮҜгғӯгғјгғ«гҒҷгӮӢгҒӢгҖҒгғҡгӮӨгғігҒ®еҸіеҒҙгӮ’гғүгғ©гғғгӮ°гҒ—гҒҰжӢЎеӨ§гҒ§гҒҚгҒҫгҒҷгҖӮ

AlwaysOn ProfilingгҒҜгҖҒгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒ®гӮігғјгғүгҒ®гӮ№гғҠгғғгғ—гӮ·гғ§гғғгғҲгҖҒгҒӨгҒҫгӮҠгӮ№гӮҝгғғгӮҜгғҲгғ¬гғјгӮ№гӮ’еёёгҒ«еҸ–еҫ—гҒ—гҒҰгҒ„гҒҫгҒҷгҖӮдҪ•еҚғгӮӮгҒ®гӮ№гӮҝгғғгӮҜгғҲгғ¬гғјгӮ№гӮ’иӘӯгҒҫгҒӘгҒ‘гӮҢгҒ°гҒӘгӮүгҒӘгҒ„гҒ“гҒЁгӮ’жғіеғҸгҒ—гҒҰгҒҝгҒҰгҒҸгҒ гҒ•гҒ„пјҒгҒқгӮҢгҒҜзҸҫе®ҹзҡ„гҒ§гҒҜгҒӮгӮҠгҒҫгҒӣгӮ“гҖӮгҒ“гӮҢгӮ’ж”ҜжҸҙгҒҷгӮӢгҒҹгӮҒгҒ«гҖҒAlwaysOn ProfilingгҒҜгғ—гғӯгғ•гӮЎгӮӨгғӘгғігӮ°гғҮгғјгӮҝгӮ’йӣҶзҙ„гҒ—гҒҰиҰҒзҙ„гҒ—гҖҒFlame GraphгҒЁе‘јгҒ°гӮҢгӮӢгғ“гғҘгғјгҒ§Call StacksгӮ’жҺўзҙўгҒҷгӮӢдҫҝеҲ©гҒӘж–№жі•гӮ’жҸҗдҫӣгҒ—гҒҫгҒҷгҖӮгҒ“гӮҢгҒҜгҖҒгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒӢгӮүгӮӯгғЈгғ—гғҒгғЈгҒ•гӮҢгҒҹгҒҷгҒ№гҒҰгҒ®гӮ№гӮҝгғғгӮҜгғҲгғ¬гғјгӮ№гҒ®иҰҒзҙ„гӮ’иЎЁгҒ—гҒҫгҒҷгҖӮFlame GraphгӮ’дҪҝз”ЁгҒ—гҒҰгҖҒгғ‘гғ•гӮ©гғјгғһгғігӮ№гҒ®е•ҸйЎҢгӮ’еј•гҒҚиө·гҒ“гҒ—гҒҰгҒ„гӮӢеҸҜиғҪжҖ§гҒ®гҒӮгӮӢгӮігғјгғүгҒ®иЎҢгӮ’зҷәиҰӢгҒ—гҖҒгӮігғјгғүгҒ«еҠ гҒҲгҒҹеӨүжӣҙгҒҢж„ҸеӣігҒ—гҒҹеҠ№жһңгӮ’жҢҒгҒЈгҒҰгҒ„гӮӢгҒӢгҒ©гҒҶгҒӢгӮ’зўәиӘҚгҒ§гҒҚгҒҫгҒҷгҖӮ

Always-on ProfilingгӮ’гҒ•гӮүгҒ«и©ігҒ—гҒҸиӘҝгҒ№гӮӢгҒ«гҒҜгҖҒMemory Stack TracesгҒ®дёӢгҒ®Profiling PaneгҒ§дёҠгҒ®з”»еғҸгҒ§еҸӮз…§гҒ•гӮҢгҒҰгҒ„гӮӢSpan **(3)**гӮ’йҒёжҠһгҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮгҒ“гӮҢгҒ«гӮҲгӮҠгҖҒAlways-on ProfilingгҒ®гғЎгӮӨгғіз”»йқўгҒҢй–ӢгҒҚгҖҒMemoryгғ“гғҘгғјгҒҢгҒӮгӮүгҒӢгҒҳгӮҒйҒёжҠһгҒ•гӮҢгҒҰгҒ„гҒҫгҒҷ

  • Timeгғ•гӮЈгғ«гӮҝгҒҜгҖҒйҒёжҠһгҒ—гҒҹгӮ№гғ‘гғігҒ®жҷӮй–“жһ гҒ«иЁӯе®ҡгҒ•гӮҢгҒҫгҒҷ (1)
  • Java Memory Metric Charts **(2)**гҒ§гҒҜгҖҒHeap MemoryгҒ®гғўгғӢгӮҝгғјгҖҒMemory Allocation Rate гӮ„ Garbage Collecting MetricsгҒӘгҒ©гҒ® Application Activity гӮ’зўәиӘҚгҒ§гҒҚгҒҫгҒҷгҖӮ
  • гӮ№гғ‘гғі **(3)**гҒ«й–ўйҖЈгҒҷгӮӢгғЎгғҲгғӘгӮҜгӮ№гҒЁStack TracesгҒ®гҒҝгҒ«гғ•гӮ©гғјгӮ«гӮ№/иЎЁзӨәгҒҷгӮӢж©ҹиғҪгҖӮгҒ“гӮҢгҒ«гӮҲгӮҠгҖҒеҝ…иҰҒгҒ«еҝңгҒҳгҒҰJavaгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒ§е®ҹиЎҢгҒ•гӮҢгҒҰгҒ„гӮӢгғҗгғғгӮҜгӮ°гғ©гӮҰгғігғүгӮўгӮҜгғҶгӮЈгғ“гғҶгӮЈгӮ’гғ•гӮЈгғ«гӮҝгҒ§йҷӨеӨ–гҒ§гҒҚгҒҫгҒҷгҖӮ
  • иӯҳеҲҘгҒ•гӮҢгҒҹJava Function calls **(4)**гҒ«гӮҲгӮҠгҖҒгҒқгҒ®й–ўж•°гҒӢгӮүе‘јгҒіеҮәгҒ•гӮҢгҒҹгғЎгӮҪгғғгғүгҒ«гғүгғӘгғ«гғҖгӮҰгғігҒ§гҒҚгҒҫгҒҷгҖӮ
  • гғ—гғӯгғ•гӮЎгӮӨгғ«гҒ•гӮҢгҒҹгӮөгғјгғ“гӮ№гҒ®гӮ№гӮҝгғғгӮҜгғҲгғ¬гғјгӮ№гҒ«еҹәгҒҘгҒҸйҡҺеұӨгҒ®иҰ–иҰҡеҢ–гӮ’жҢҒгҒӨFlame Graph (5)гҖӮ
  • гӮөгғјгғ“гӮ№гҒҢиӨҮж•°гҒ®гғҗгғјгӮёгғ§гғігӮ’иө·еӢ•гҒҷгӮӢе ҙеҗҲгҒ«еӮҷгҒҲгҒҰгҖҒService instance **(6)**гӮ’йҒёжҠһгҒҷгӮӢж©ҹиғҪгҖӮ

гҒ•гӮүгҒӘгӮӢиӘҝжҹ»гҒ®гҒҹгӮҒгҒ«гҖҒUIгҒ§гҒҜгӮ№гӮҝгғғгӮҜгғҲгғ¬гғјгӮ№гӮ’гӮҜгғӘгғғгӮҜгҒ—гҒҰгҖҒе‘јгҒіеҮәгҒ•гӮҢгҒҹй–ўж•°гҒЁгҖҒгғ•гғ¬гғјгғ гғҒгғЈгғјгғҲгҒӢгӮүй–ўйҖЈгҒҷгӮӢиЎҢгӮ’зўәиӘҚгҒ§гҒҚгҒҫгҒҷгҖӮгҒ“гӮҢгӮ’дҪҝз”ЁгҒ—гҒҰгҖҒгӮігғјгғҮгӮЈгғігӮ°гғ—гғ©гғғгғҲгғ•гӮ©гғјгғ гҒ§е®ҹйҡӣгҒ®гӮігғјгғүгҒ®иЎҢгӮ’иЎЁзӨәгҒ§гҒҚгҒҫгҒҷпјҲгӮӮгҒЎгӮҚгӮ“гҖҒгҒҠеҘҪгҒҝгҒ®гӮігғјгғҮгӮЈгғігӮ°гғ—гғ©гғғгғҲгғ•гӮ©гғјгғ гҒ«гӮҲгҒЈгҒҰз•°гҒӘгӮҠгҒҫгҒҷпјүгҖӮ

Database Query Performance

Database Query PerformanceгӮ’дҪҝз”ЁгҒҷгӮӢгҒЁгҖҒSplunk APMгҒ§зӣҙжҺҘгҖҒгғҮгғјгӮҝгғҷгғјгӮ№гӮҜгӮЁгғӘгҒҢгӮөгғјгғ“гӮ№гҒ®еҸҜз”ЁжҖ§гҒ«дёҺгҒҲгӮӢеҪұйҹҝгӮ’гғўгғӢгӮҝгғјгҒ§гҒҚгҒҫгҒҷгҖӮгҒ“гӮҢгҒ«гӮҲгӮҠгҖҒгғҮгғјгӮҝгғҷгғјгӮ№гӮ’гӮӨгғігӮ№гғҲгғ«гғЎгғігғҲгҒҷгӮӢгҒ“гҒЁгҒӘгҒҸгҖҒй•·жҷӮй–“е®ҹиЎҢгҒ•гӮҢгӮӢгӮҜгӮЁгғӘгҖҒжңҖйҒ©еҢ–гҒ•гӮҢгҒҰгҒ„гҒӘгҒ„гӮҜгӮЁгғӘгҖҒгҒҫгҒҹгҒҜйҮҚгҒ„гӮҜгӮЁгғӘгӮ’иҝ…йҖҹгҒ«зү№е®ҡгҒ—гҖҒгҒқгӮҢгӮүгҒҢеј•гҒҚиө·гҒ“гҒ—гҒҰгҒ„гӮӢеҸҜиғҪжҖ§гҒ®гҒӮгӮӢе•ҸйЎҢгӮ’и»ҪжёӣгҒ§гҒҚгҒҫгҒҷгҖӮ

гғҮгғјгӮҝгғҷгғјгӮ№гӮҜгӮЁгғӘгҒ®гғ‘гғ•гӮ©гғјгғһгғігӮ№гӮ’зўәиӘҚгҒҷгӮӢгҒ«гҒҜгҖҒгғ–гғ©гӮҰгӮ¶гҒ§жҲ»гӮӢгҒӢгҖҒгғЎгғӢгғҘгғјгғҗгғјгҒ®APMгӮ»гӮҜгӮ·гғ§гғігҒ«з§»еӢ•гҒ—гҒҰAPMгҒ®Service MapгғҡгғјгӮёгҒ«з§»еӢ•гҒ—гҖҒService MapгӮҝгӮӨгғ«гӮ’гӮҜгғӘгғғгӮҜгҒ—гҒҫгҒҷгҖӮ

Dependency mapгҒ§жҺЁи«–гҒ•гӮҢгҒҹгғҮгғјгӮҝгғҷгғјгӮ№гӮөгғјгғ“гӮ№ mysql:petclinic Inferred Database serverгӮ’йҒёжҠһгҒ— (1)гҖҒж¬ЎгҒ«еҸіеҒҙгҒ®гғҡгӮӨгғігӮ’гӮ№гӮҜгғӯгғјгғ«гҒ—гҒҰDatabase Query Performance Pane **(2)**гӮ’иҰӢгҒӨгҒ‘гҒҫгҒҷгҖӮ

гғһгғғгғ—гҒ§йҒёжҠһгҒ—гҒҹгӮөгғјгғ“гӮ№гҒҢе®ҹйҡӣгҒ«пјҲжҺЁи«–гҒ•гӮҢгҒҹпјүгғҮгғјгӮҝгғҷгғјгӮ№гӮөгғјгғҗгғјгҒ§гҒӮгӮӢе ҙеҗҲгҖҒгҒ“гҒ®гғҡгӮӨгғігҒ«гҒҜжңҹй–“гҒ«еҹәгҒҘгҒҸдёҠдҪҚ90%пјҲP90пјүгҒ®гғҮгғјгӮҝгғҷгғјгӮ№гӮігғјгғ«гҒҢиЎЁзӨәгҒ•гӮҢгҒҫгҒҷгҖӮdb-queryгғ‘гғ•гӮ©гғјгғһгғігӮ№ж©ҹиғҪгӮ’гҒ•гӮүгҒ«и©ігҒ—гҒҸиӘҝгҒ№гӮӢгҒ«гҒҜгҖҒгғҡгӮӨгғігҒ®дёҠйғЁгҒ«гҒӮгӮӢDatabase Query PerformanceгҒЁгҒ„гҒҶеҚҳиӘһгҒ®гҒ©гҒ“гҒӢгӮ’гӮҜгғӘгғғгӮҜгҒ—гҒҫгҒҷгҖӮ

гҒ“гӮҢгҒ«гӮҲгӮҠгҖҒDB-query PerformanceгҒ®жҰӮиҰҒз”»йқўгҒҢиЎЁзӨәгҒ•гӮҢгҒҫгҒҷ

Database Query Normalization

гғҮгғ•гӮ©гғ«гғҲгҒ§гҒҜгҖҒSplunk APMгӮӨгғігӮ№гғҲгғ«гғЎгғігғҶгғјгӮ·гғ§гғігҒҜгғҮгғјгӮҝгғҷгғјгӮ№гӮҜгӮЁгғӘгӮ’гӮөгғӢгӮҝгӮӨгӮәгҒ—гҒҰгҖҒdb.statements гҒӢгӮүгӮ·гғјгӮҜгғ¬гғғгғҲгӮ„еҖӢдәәгӮ’зү№е®ҡгҒ§гҒҚгӮӢжғ…е ұпјҲPIIпјүгҒӘгҒ©гҒ®ж©ҹеҜҶгғҮгғјгӮҝгӮ’еүҠйҷӨгҒҫгҒҹгҒҜгғһгӮ№гӮҜгҒ—гҒҫгҒҷгҖӮгғҮгғјгӮҝгғҷгғјгӮ№гӮҜгӮЁгғӘгҒ®жӯЈиҰҸеҢ–гӮ’гӮӘгғ•гҒ«гҒҷгӮӢж–№жі•гҒҜгҒ“гҒЎгӮүгҒ§зўәиӘҚгҒ§гҒҚгҒҫгҒҷгҖӮ

гҒ“гҒ®з”»йқўгҒ«гҒҜгҖҒSplunk Observability CloudгҒ«йҖҒдҝЎгҒ•гӮҢгҒҹTraces & SpansгҒ«еҹәгҒҘгҒ„гҒҰгҖҒгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒӢгӮүгғҮгғјгӮҝгғҷгғјгӮ№гҒ«еҜҫгҒ—гҒҰе®ҹиЎҢгҒ•гӮҢгҒҹгҒҷгҒ№гҒҰгҒ®Database queries **(1)**гҒҢиЎЁзӨәгҒ•гӮҢгҒҫгҒҷгҖӮжҷӮй–“гғ–гғӯгғғгӮҜй–“гҒ§жҜ”ијғгҒ—гҒҹгӮҠгҖҒTotal TimeгҖҒP90 Latency & Requests **(2)**гҒ§гӮҪгғјгғҲгҒ—гҒҹгӮҠгҒ§гҒҚгӮӢгҒ“гҒЁгҒ«жіЁж„ҸгҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮ

гғӘгӮ№гғҲеҶ…гҒ®еҗ„Database queryгҒ«гҒӨгҒ„гҒҰгҖҒжҷӮй–“гӮҰгӮЈгғігғүгӮҰдёӯгҒ®жңҖй«ҳгғ¬гӮӨгғҶгғігӮ·гҖҒгӮігғјгғ«гҒ®з·Ҹж•°гҖҒгҒҠгӮҲгҒі1з§’гҒӮгҒҹгӮҠгҒ®гғӘгӮҜгӮЁгӮ№гғҲж•° **(3)**гҒҢиЎЁзӨәгҒ•гӮҢгҒҫгҒҷгҖӮгҒ“гӮҢгҒ«гӮҲгӮҠгҖҒгӮҜгӮЁгғӘгӮ’жңҖйҒ©еҢ–гҒ§гҒҚгӮӢе ҙжүҖгӮ’зү№е®ҡгҒ§гҒҚгҒҫгҒҷгҖӮ

еҸіеҒҙгҒ®гғҡгӮӨгғі **(5)**гҒ®2гҒӨгҒ®гғҒгғЈгғјгғҲгӮ’дҪҝз”ЁгҒ—гҒҰгҖҒDatabase CallsгӮ’еҗ«гӮҖгғҲгғ¬гғјгӮ№гӮ’йҒёжҠһгҒ§гҒҚгҒҫгҒҷгҖӮTag SpotlightгғҡгӮӨгғі **(6)**гӮ’дҪҝз”ЁгҒ—гҒҰгҖҒгӮЁгғігғүгғқгӮӨгғігғҲгӮ„гӮҝгӮ°гҒ«еҹәгҒҘгҒ„гҒҰгҖҒгғҮгғјгӮҝгғҷгғјгӮ№гӮігғјгғ«гҒ«й–ўйҖЈгҒҷгӮӢгӮҝгӮ°гӮ’зўәиӘҚгҒ—гҒҫгҒҷгҖӮ

гӮҜгӮЁгғӘгҒ®и©ізҙ°гғ“гғҘгғјгӮ’иЎЁзӨәгҒҷгӮӢеҝ…иҰҒгҒҢгҒӮгӮӢе ҙеҗҲ

зү№е®ҡгҒ®Query **(1)**гӮ’гӮҜгғӘгғғгӮҜгҒ—гҒҫгҒҷгҖӮгҒ“гӮҢгҒ«гӮҲгӮҠгҖҒQuery Details pane **(2)**гҒҢй–ӢгҒҚгҖҒгӮҲгӮҠи©ізҙ°гҒӘиӘҝжҹ»гҒ«дҪҝз”ЁгҒ§гҒҚгҒҫгҒҷгҖӮ
